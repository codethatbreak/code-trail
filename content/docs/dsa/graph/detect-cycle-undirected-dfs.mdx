---
title: Detect Cycle in Undirected Graph Using DFS
---

### Concept and Intuition

- **Definition of a Cycle:** A cycle exists if you can start from a node and return to it via any path without breaking or repeating an edge.
- **The DFS Approach:** The algorithm explores as far as possible along each branch before backtracking. If, during this deep traversal, you encounter a node that has **already been visited** and is **not the parent** of the current node, a cycle is confirmed.
- **The "Parent" Node:** In an undirected graph, every edge is bi-directional. If you move from node 1 to node 2, node 2 will see node 1 as a visited neighbor. To avoid incorrectly identifying this back-link as a cycle, you must track the node you just came from (the **parent**).

### The DFS Algorithm Logic

1.  **Initialization:** Create a **visited array** (initialized to 0) to track traversed nodes.
2.  **Recursive Call:** Call the DFS function, passing the **current node** and its **parent** (for the starting node, the parent is typically -1).
3.  **Marking:** Upon entering a node, mark it as visited.
4.  **Neighbor Exploration:** Iterate through all neighbors in the adjacency list:
    - **If the neighbor is unvisited:** Recursively call DFS for that neighbor, setting the **current node as the parent**. If any of these recursive calls return `true`, a cycle exists elsewhere in that path, so return `true` immediately.
    - **If the neighbor is already visited:**
      - Check if the neighbor is the **parent**. If it is, ignore it and continue.
      - If the neighbor is **not the parent**, you have encountered a node that was previously visited in the path by a different route. This is a **cycle**; return `true`.
5.  **Termination:** If all neighbors are explored without finding a cycle, return `false`.
6.  **Multiple Components:** To handle disconnected graphs, wrap the initial call in a loop from 1 to $N$. If a node is unvisited, trigger the DFS. If any component returns `true`, the entire graph is considered to have a cycle.

### Implementation Code

#### C++ Implementation

```cpp
bool dfs(int node, int parent, int vis[], vector<int> adj[]) {
    vis[node] = 1; // Mark current node as visited
    for (auto it : adj[node]) {
        if (!vis[it]) {
            // Recurse for unvisited neighbors, passing current node as parent
            if (dfs(it, node, vis, adj)) return true;
        }
        // Cycle condition: neighbor is visited AND is not the parent
        else if (it != parent) return true;
    }
    return false;
}
```

### Complexity Analysis

- **Time Complexity:** **$O(N + 2E)$**. Each node is visited once ($N$), and the algorithm iterates through the degrees of all nodes ($2E$ for an undirected graph). There is also a small $O(N)$ overhead for the outer loop used to handle multiple components.
- **Space Complexity:** **$O(N)$**. This accounts for the visited array and the **recursion stack space**, which in the worst-case (a linear, skewed graph) can reach $N$.
