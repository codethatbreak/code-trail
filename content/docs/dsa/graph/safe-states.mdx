### Core Definitions

- **Terminal Node:** A node is considered terminal if it has **no outgoing edges** (its out-degree is zero).
- **Safe Node:** A node is safe if **every possible path** starting from that node eventually leads to a terminal node.
- **The Objective:** Identify all safe nodes in a directed graph and return them in **sorted ascending order**.

### Key Intuition: The Cycle Connection

The problem of finding safe nodes is essentially a variation of **cycle detection** in a directed graph:

- **Cycles are Unsafe:** Any node that is **part of a cycle** can never be a safe node because it has at least one path that loops infinitely and never reaches a terminal node.
- **Leading to Cycles is Unsafe:** Any node that has a **path leading into a cycle** is also unsafe.
- **The Safe Condition:** Any node that is **not part of a cycle** and **does not lead to a cycle** will eventually end up at a terminal node and is therefore a safe node.

### The DFS Algorithm Logic

The solution uses the standard **directed graph cycle detection** logic involving two tracking arrays, plus a third array to mark the final status of each node.

#### **Data Structures Required:**

1.  **Visited Array (`vis`):** Tracks if a node has been explored during the entire traversal.
2.  **Path Visited Array (`pathVis`):** Tracks if a node is currently in the **active recursion stack** (the current path).
3.  **Check Array (`check`):** A boolean or integer array used to mark which nodes are confirmed as **safe nodes**.

#### **Step-by-Step Process:**

1.  Initialize `vis`, `pathVis`, and `check` arrays to 0.
2.  Iterate through every node (0 to $V-1$) to handle multiple components. If a node is unvisited, call the DFS function.
3.  **Inside the DFS function for a node:**
    - Mark the node as **visited** and **path visited**.
    - Initially, assume the node might be part of a cycle (keep its `check` status as 0).
    - Iterate through all neighbors of the current node:
      - **If the neighbor is unvisited:** Recursively call DFS. If that call returns `true` (a cycle is found), the current node is unsafe—return `true` immediately.
      - **If the neighbor is already path visited:** A cycle has been detected—return `true`.
    - **If the loop finishes** and no cycle was found in any path, it means the node is safe.
    - **Unmark path visited** (set back to 0) and mark the node in the **check array as 1** (Safe).
    - Return `false` (no cycle found for this path).
4.  After the entire traversal is complete, iterate through the `check` array and collect all nodes marked as 1 into the final result list.

### Implementation Code

#### C++

```cpp
bool dfsCheck(int node, vector<int> adj[], int vis[], int pathVis[], int check[]) {
    vis[node] = 1;
    pathVis[node] = 1;
    check[node] = 0; // Initially assume unsafe

    for (auto it : adj[node]) {
        // When neighbor is unvisited
        if (!vis[it]) {
            if (dfsCheck(it, adj, vis, pathVis, check)) return true;
        }
        // If neighbor is visited and on the same path, cycle detected
        else if (pathVis[it]) {
            return true;
        }
    }

    // If no cycle found, mark as safe and backtrack
    check[node] = 1;
    pathVis[node] = 0;
    return false;
}

vector<int> eventualSafeNodes(int V, vector<int> adj[]) {
    int vis[V] = {0}, pathVis[V] = {0}, check[V] = {0};
    vector<int> safeNodes;

    for (int i = 0; i < V; i++) {
        if (!vis[i]) {
            dfsCheck(i, adj, vis, pathVis, check);
        }
    }

    for (int i = 0; i < V; i++) {
        if (check[i] == 1) safeNodes.push_back(i);
    }
    return safeNodes;
}
```

### Complexity Analysis

- **Time Complexity:** **$O(V + E)$**. This is consistent with a standard DFS traversal where every node and every directed edge is visited exactly once.
- **Space Complexity:** **$O(3N)$** simplified to **$O(N)$**. This accounts for the `visited`, `pathVisited`, and `check` arrays, along with the recursion stack space, which can be up to $N$ in a skewed graph.
