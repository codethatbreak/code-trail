---
title: Kruskal's Algorithm
---

**Kruskal’s Algorithm** is a popular method for finding the **Minimum Spanning Tree (MST)** of a given graph. Unlike Prim's algorithm, which grows the MST from a single starting node, Kruskal's is an edge-based approach that utilizes the **Disjoint Set Union (DSU)** data structure.

### Core Objective

The algorithm aims to find a subset of edges that connect all **$N$ nodes** using exactly **$N-1$ edges** such that the total weight is minimized and no cycles are formed.

### The Step-by-Step Process

1.  **Extract and Sort Edges:** If the input is an adjacency list, first convert it into an edge list where each entry is a triplet: **`(weight, node1, node2)`**. **Sort all edges** in non-decreasing order according to their weights.
2.  **Initialize DSU:** Create a Disjoint Set for all $N$ nodes. Initially, every node is its own "ultimate parent" or boss.
3.  **Iterate Through Sorted Edges:** For each edge in the sorted list:
    - Find the **ultimate parent** of the two nodes connected by the edge.
    - **Check for Cycles:**
      - If the ultimate parents are **different**, the nodes belong to different components. **Include this edge** in the MST, add its weight to the total sum, and perform a **Union** (by rank or size) to merge the two components.
      - If the ultimate parents are the **same**, the nodes are already connected. **Discard the edge** to avoid creating a cycle.
4.  **Completion:** The process continues until all edges are processed or $N-1$ edges are added to the MST.

### Implementation Logic (C++ Style)

To implement this, you must integrate the `DisjointSet` class (with `findUPar` and `unionBySize/Rank` functions) into your main algorithm.

```cpp
int spanningTree(int V, vector<vector<int>> adj[]) {
    // 1. Convert adjacency list to edge list and sort
    vector<pair<int, pair<int, int>>> edges;
    for (int i = 0; i < V; i++) {
        for (auto it : adj[i]) {
            int adjNode = it;
            int wt = it;
            edges.push_back({wt, {i, adjNode}});
        }
    }
    sort(edges.begin(), edges.end());

    // 2. Initialize DSU and MST weight
    DisjointSet ds(V);
    int mstWt = 0;

    // 3. Process edges
    for (auto it : edges) {
        int wt = it.first;
        int u = it.second.first;
        int v = it.second.second;

        // If nodes belong to different components, union them
        if (ds.findUPar(u) != ds.findUPar(v)) {
            mstWt += wt;
            ds.unionBySize(u, v);
        }
    }
    return mstWt;
}
```

### Complexity Analysis

- **Time Complexity:** **$O(M \log M + M \times 4\alpha)$**, where $M$ is the number of edges.
  - $O(M \log M)$ is required to **sort the edges**.
  - $O(M \times 4\alpha)$ covers the $M$ iterations of DSU operations (find and union), which are nearly constant.
- **Space Complexity:** **$O(M + N)$**.
  - $O(M)$ to store the edge list.
  - $O(N)$ for the parent and rank/size arrays within the DSU data structure.

### Key Difference: Kruskal's vs. Prim's

While both find the MST, Kruskal's is generally easier to implement if you already have a DSU class ready. It is particularly efficient for **sparse graphs** because it focuses on edges, whereas Prim's is often preferred for **dense graphs**.
