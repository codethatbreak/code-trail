## Definition and Concept

Given a **non-empty array** where every element appears **exactly three times**, except for one element that appears exactly once, find that single unique element. The solution must run in linear time $O(n)$ with constant space $O(1)$.

### Intuition — How to Think About This

In [Single Number I](./single-number-i), XOR worked because pairs cancel out (`x ^ x = 0`). But here, elements appear **three times**, not twice. XOR doesn't help directly: `x ^ x ^ x = x`, so the triplets don't cancel!

**Key insight**: Instead of working with individual bits as binary (0 or 1), we count how many numbers have a `1` at each bit position. For every bit position:
- If the count is divisible by 3, the unique element has a `0` there
- If the count leaves remainder 1 when divided by 3, the unique element has a `1` there

This works because the triplet elements contribute multiples of 3 to each bit count.

### Visual Representation — Bit Counting Approach

```
Array: [2, 2, 3, 2] (binary: 010, 010, 011, 010)

Bit position counts:
Position 0 (LSB): 0 + 0 + 1 + 0 = 1 → 1 % 3 = 1 → unique bit is 1
Position 1:        1 + 1 + 1 + 1 = 4 → 4 % 3 = 1 → unique bit is 1
Position 2 (MSB):  0 + 0 + 0 + 0 = 0 → 0 % 3 = 0 → unique bit is 0

Result: bits [0, 1, 1] reading from MSB to LSB = 011 = 3 ✓
```

### Alternative Approach — Bucket Method (from the video)

The video presents an elegant approach using two "buckets" that track which elements have appeared once and twice:

- **Bucket one**: Elements seen exactly once so far
- **Bucket two**: Elements seen exactly twice so far

When a number enters bucket two, it's removed from bucket one (if present). When a number appears three times, it's removed from both buckets. At the end, only the unique element remains in bucket one.

```
Array: [2, 3, 2, 4, 2, 3, 3]

Processing each number:

Number | Action                              | Bucket One | Bucket Two
-------|-------------------------------------|------------|----------
   2   | First time → add to bucket one      | {2}        | {}
   3   | First time → add to bucket one      | {2, 3}     | {}
   2   | Second time → remove from B1, add to B2 | {3}    | {2}
   4   | First time → add to bucket one      | {3, 4}     | {2}
   2   | Third time → remove from both       | {3, 4}     | {}
   3   | Second time → remove from B1, add to B2 | {4}    | {3}
   3   | Third time → remove from both       | {4}        | {}

Final: Bucket one = {4}, Bucket two = {}
Answer: 4 ✓
```

### Step-by-Step Thinking Process (Bucket Method)

1. **What do we need to track?** Which elements have appeared once vs. twice
2. **How does a number move between states?** 
   - First appearance → goes to bucket one
   - Second appearance → removed from bucket one, added to bucket two
   - Third appearance → removed from both buckets (cancels out)
3. **Why does this work?** Triplets cancel completely; only the unique element survives in bucket one

### The Algorithm / Step-by-Step Logic (Bucket Method)

```
Given array nums:

Step 1: Initialize b1 = 0, b2 = 0 (two buckets)
Step 2: For each number num in nums:
        a. If num is already in b1, remove it from b1 and add to b2
           → b1 = b1 ^ num; b2 = b2 ^ num
        b. Else if num is in b2, remove it from both (third occurrence)
           → Clear bits that are in both b1 and b2: 
              common = b1 & b2
              b1 &= ~common; b2 &= ~common
Step 3: Return b1 (contains the unique element)
```

### Approach 1: Bit Counting (Clearer and More Common)

#### The Algorithm

```
Step 1: Initialize result = 0
Step 2: For each bit position i from 0 to 31:
        a. Count how many numbers have the i-th bit set
        b. If count % 3 == 1, set the i-th bit in result
Step 3: Return result
```

#### Worked Example — Step by Step (Bit Counting)

```
Array: [2, 2, 3, 2] → binary: [010, 010, 011, 010]

Bit position 0 (LSB):
  Bit values: 0 + 0 + 1 + 0 = 1
  1 % 3 = 1 → set bit 0 in result
  result so far: ...001

Bit position 1:
  Bit values: 1 + 1 + 1 + 1 = 4
  4 % 3 = 1 → set bit 1 in result
  result so far: ...011

Bit position 2 (MSB):
  Bit values: 0 + 0 + 0 + 0 = 0
  0 % 3 = 0 → don't set bit 2
  result so far: ...011

Final result: 011 = 3 ✓
```

### Complexity Analysis

- **Time Complexity**: $O(32 \times n) = O(n)$ — for each of the 32 bits, we scan all n elements
- **Space Complexity**: $O(1)$ — only a few integer variables

### Implementation Code — C++ (Bit Counting Approach)

```cpp
#include <iostream>
#include <vector>
using namespace std;

int singleNumberII(vector<int>& nums) {
    int result = 0;
    
    // Check each bit position (32-bit integer)
    for (int i = 0; i < 32; i++) {
        int count = 0;
        
        // Count how many numbers have the i-th bit set
        for (int num : nums) {
            if ((num >> i) & 1) {
                count++;
            }
        }
        
        // If count % 3 == 1, the unique number has this bit set
        if (count % 3 == 1) {
            result |= (1 << i);  // Set the i-th bit in result
        }
    }
    
    return result;
}

int main() {
    vector<int> test = [2, 2, 3, 2];
    cout << "Single number: " << singleNumberII(test) << endl;  // Output: 3
    
    return 0;
}
```

### Key Logic Summary / Important Points

- **Bit counting approach**: Count set bits at each position across all numbers. The unique element's bits are those where count % 3 = 1
- **Why it works**: Triplet elements contribute multiples of 3 to each bit count, so only the unique element's bits leave a remainder of 1
- **Time complexity**: $O(32n) = O(n)$ — constant factor of 32 for 32-bit integers
- **Space complexity**: $O(1)$ — fixed number of variables regardless of input size
- **Negative numbers**: The bit counting approach handles negative numbers correctly in two's complement representation
- **Alternative approaches exist** using finite state machines with two variables (b1, b2) that track which elements have appeared once vs. twice
