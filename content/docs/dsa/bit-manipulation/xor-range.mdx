## Definition and Concept

Given two integers `left` and `right`, compute the **XOR of all numbers from left to right** (inclusive): `left ^ (left+1) ^ ... ^ right`.

### Intuition — How to Think About This

Computing XOR for every number in a range would take $O(right - left)$ time. But there's an $O(1)$ pattern-based approach!

**Key insight**: The XOR of numbers from 0 to n follows a repeating pattern with period 4:

| n % 4 | XOR(0, 1, ..., n) |
|-------|-------------------|
| 0     | n                 |
| 1     | 1                 |
| 2     | n + 1             |
| 3     | 0                 |

Once we can compute `xor(0, right)` and `xor(0, left - 1)`, the answer is simply:

$$\text{xor}(left, right) = \text{xor}(0, right) \oplus \text{xor}(0, left-1)$$

This works because XOR has the cancellation property: elements appearing in both ranges cancel out.

```
Pattern repeats every 4! ✓
```

### Step-by-Step Thinking Process

1. **What's the brute force?** XOR all numbers from left to right — $O(right - left)$ time
2. **Can we do better?** Yes — find a pattern in `xor(0, n)` for various values of n
3. **The pattern**: depends on `n % 4` with period 4 (values: n, 1, n+1, 0)
4. **Range XOR formula**: `xor(left, right) = xor(0, right) ^ xor(0, left-1)` — elements from 0 to left-1 cancel out

### The Algorithm / Step-by-Step Logic

```
Given left and right:

Step 1: Define helper function computeXor(n):
        a. Compute remainder = n % 4
        b. If remainder == 0 → return n
        c. If remainder == 1 → return 1
        d. If remainder == 2 → return n + 1
        e. If remainder == 3 → return 0

Step 2: Compute answer = computeXor(right) ^ computeXor(left - 1)

Step 3: Return answer ✓
```

### Worked Example — Step by Step

#### Example 1: left = 5, right = 8

```
Expected: 5 ^ 6 ^ 7 ^ 8 = ?

Manual computation:
  5 = 0101
  6 = 0110
  7 = 0111
  8 = 1000
  
  5 ^ 6 = 3 (0101 ^ 0110 = 0011)
  3 ^ 7 = 4 (0011 ^ 0111 = 0100)
  4 ^ 8 = 12 (0100 ^ 1000 = 1100)

Answer: 12 ✓

Using the formula:
Step 1: computeXor(8):
  8 % 4 = 0 → return 8
  
Step 2: computeXor(4):
  4 % 4 = 0 → return 4
  
Step 3: answer = 8 ^ 4 = 12 ✓✓

Verification: 5 ^ 6 ^ 7 ^ 8 = (0^1^2^3^4^5^6^7^8) ^ (0^1^2^3^4)
            = computeXor(8) ^ computeXor(4)
            = 8 ^ 4
            = 12 ✓
```

#### Example 2: left = 1, right = 3

```
Expected: 1 ^ 2 ^ 3 = ?

Manual computation:
  1 ^ 2 = 3 (01 ^ 10 = 11)
  3 ^ 3 = 0 (11 ^ 11 = 00)

Answer: 0 ✓

Using the formula:
Step 1: computeXor(3):
  3 % 4 = 3 → return 0
  
Step 2: computeXor(0):
  0 % 4 = 0 → return 0
  
Step 3: answer = 0 ^ 0 = 0 ✓✓

Verification: 1 ^ 2 ^ 3 = (0^1^2^3) ^ (0) = 0 ^ 0 = 0 ✓
```

#### Example 3: left = 8, right = 10

```
Expected: 8 ^ 9 ^ 10 = ?

Manual computation:
  8 = 1000
  9 = 1001
  10 = 1010
  
  8 ^ 9 = 1 (1000 ^ 1001 = 0001)
  1 ^ 10 = 11 (0001 ^ 1010 = 1011)

Answer: 11 ✓

Using the formula:
Step 1: computeXor(10):
  10 % 4 = 2 → return 10 + 1 = 11
  
Step 2: computeXor(7):
  7 % 4 = 3 → return 0
  
Step 3: answer = 11 ^ 0 = 11 ✓✓

Verification: 8 ^ 9 ^ 10 = (0^...^10) ^ (0^...^7) = 11 ^ 0 = 11 ✓
```

### Complexity Analysis

- **Time Complexity**: $O(1)$ — constant number of operations regardless of input size
- **Space Complexity**: $O(1)$ — no extra space needed

### Implementation Code — C++

```cpp
#include <iostream>

// Compute XOR from 0 to n using the pattern
int computeXor(int n) {
    int remainder = n % 4;
    
    if (remainder == 0) return n;       // Pattern: n
    if (remainder == 1) return 1;        // Pattern: 1
    if (remainder == 2) return n + 1;   // Pattern: n+1
    return 0;                            // Pattern: 0
}

// Compute XOR of all numbers from left to right
int xorInRange(int left, int right) {
    return computeXor(right) ^ computeXor(left - 1);
}

int main() {
    // Test cases
    std::cout << "XOR(5, 8) = " << xorInRange(5, 8) << endl;   // Output: 12
    std::cout << "XOR(1, 3) = " << xorInRange(1, 3) << endl;   // Output: 0
    std::cout << "XOR(8, 10) = " << xorInRange(8, 10) << endl; // Output: 11
    
    return 0;
}
```

### Key Logic Summary / Important Points

- **Pattern of XOR(0 to n)** repeats every 4 values with period: `[n, 1, n+1, 0]` based on `n % 4`
- **Range XOR formula**: `xor(left, right) = xor(0, right) ^ xor(0, left - 1)` — cancellation property of XOR
- **$O(1)$ solution** — no iteration needed! This is dramatically faster than the brute force $O(right - left)$ approach for large ranges
- **Edge case**: When `left = 0`, `computeXor(left - 1) = computeXor(-1)` which needs special handling. In practice, if `left = 0`, just return `computeXor(right)` directly
- **Why the pattern works**: XOR of consecutive numbers has a periodic structure because each bit position toggles at regular intervals (bit 0 toggles every number, bit 1 toggles every 2 numbers, etc.)
