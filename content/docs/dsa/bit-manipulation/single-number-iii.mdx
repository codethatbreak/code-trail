title: "Single Number III — Finding Two Unique Elements"
---

## Definition and Concept

Given a **non-empty array** where exactly two elements appear **exactly once**, and all other elements appear **exactly twice**, find those two unique elements. The solution must run in linear time $O(n)$ with constant space $O(1)$.

### Intuition — How to Think About This

In [Single Number I](./single-number-i), XORing everything gave us the unique element because pairs canceled out. Here, if we XOR all elements, we get `a ^ b` (where a and b are our two unique numbers). But this doesn't directly tell us what a and b are individually!

**Key insight**: Since `a ≠ b`, their XOR result must have at least one bit set to 1. This differing bit tells us: **at this position, one of the numbers has a 0 and the other has a 1**. We can use this to partition the array into two groups and solve two Single Number I problems!

### Step-by-Step Thinking Process

1. **XOR all elements** → gives us `a ^ b` (the two unique numbers XORed together)
2. **Find a differing bit** between `a` and `b` — use the rightmost set bit of `(a ^ b)`
3. **Partition the array** into two groups based on whether that bit is 0 or 1
4. **XOR each group separately** → each group now has exactly one unique element (the pairs cancel out within each group)

### The Algorithm / Step-by-Step Logic

```
Given array nums:

Step 1: XOR all elements → xorAll = a ^ b
Step 2: Find rightmost set bit of xorAll: diff = xorAll & (-xorAll)
        (or equivalently: diff = xorAll & (~xorAll + 1))
Step 3: Initialize x = 0, y = 0 (two accumulators for the two groups)
Step 4: For each number num in nums:
        a. If (num & diff) == 0 → XOR into x
        b. Else → XOR into y
Step 5: Return [x, y] as the two unique elements ✓
```

### Worked Example — Step by Step

#### Example: nums = [1, 2, 1, 3, 2, 5]

```
Step 1: XOR all elements
  xorAll = 1 ^ 2 ^ 1 ^ 3 ^ 2 ^ 5
         = (1^1) ^ (2^2) ^ 3 ^ 5
         = 0 ^ 0 ^ 3 ^ 5
         = 3 ^ 5
         = 011 ^ 101
         = 110 (= 6)

Step 2: Find rightmost set bit of 6 (binary: 110)
  diff = 6 & (-6)
  
  First, find -6 in two's complement:
    +6 = 000...0110
    ~6 = 111...1001
    ~6 + 1 = 111...1010
  
  diff = 000...0110 & 111...1010 = 000...0010 (= 2)

Step 3: Initialize x = 0, y = 0

Step 4: Partition and XOR each group using mask = 2 (binary: ...0010)

  Number 1 (binary: 001):
    (1 & 2) = 0 → Group A
    x = 0 ^ 1 = 1
  
  Number 2 (binary: 010):
    (2 & 2) = 2 ≠ 0 → Group B
    y = 0 ^ 2 = 2
  
  Number 1 (binary: 001):
    (1 & 2) = 0 → Group A
    x = 1 ^ 1 = 0
  
  Number 3 (binary: 011):
    (3 & 2) = 2 ≠ 0 → Group B
    y = 2 ^ 3 = 1
  
  Number 2 (binary: 010):
    (2 & 2) = 2 ≠ 0 → Group B
    y = 1 ^ 2 = 3
  
  Number 5 (binary: 101):
    (5 & 2) = 0 → Group A
    x = 0 ^ 5 = 5

Step 5: Return [x, y] = [5, 3] ✓✓
```

### Complexity Analysis

- **Time Complexity**: $O(n)$ — two passes through the array (one for XOR all, one for partitioning)
- **Space Complexity**: $O(1)$ — only a few integer variables used

### Implementation Code — C++

```cpp
#include <iostream>
#include <vector>
using namespace std;

// Find the two unique elements in an array where every other element appears twice
vector<int> singleNumberIII(vector<int>& nums) {
    // Step 1: XOR all elements → gives a ^ b
    int xorAll = 0;
    for (int num : nums) {
        xorAll ^= num;
    }
    
    // Step 2: Find rightmost set bit of (a ^ b)
    // This is the first bit where a and b differ
    int diff = xorAll & (-xorAll);
    
    // Step 3: Partition into two groups and XOR each group
    int x = 0, y = 0;
    for (int num : nums) {
        if ((num & diff) == 0) {
            x ^= num;  // Group A: bit is 0
        } else {
            y ^= num;  // Group B: bit is 1
        }
    }
    
    return {x, y};
}

int main() {
    vector<int> test = [1, 2, 1, 3, 2, 5];
    auto result = singleNumberIII(test);
    cout << "Unique elements: " << result[0] << ", " << result[1] << endl;
    // Output: Unique elements: 5, 3 (order may vary)
    
    return 0;
}
```

### Key Logic Summary / Important Points

- **XOR all → get a ^ b**, then find a differing bit to partition the array
- **Rightmost set bit**: `diff = xorAll & (-xorAll)` — this isolates the rightmost 1-bit using two's complement trick
- **Partitioning works because** pairs of identical numbers always go into the same group (they have the same bits), so they cancel within each group
- **The differing bit guarantees separation**: Since `a` and `b` differ at this position, one goes to Group A and the other to Group B
- **Edge case**: If the array has only two elements, both are unique — XOR all gives their XOR, then partition correctly separates them

- [Bit Manipulation Tricks](./bit-manipulation-tricks) — Rightmost set bit trick (`n & -n`)
